## Production Access

Production access will be given after you have submitted all required document and pass KYC/KYB process from our end and CEX side. Once the onboarding is approved, we will initiate the credentials sharing with the details shared in the preparation.

Expect 1 week working days SLA after document submission if there are no feedback for the document from provider/ partner.

- Credentials needed to go live:
  - Credentials shared from our side (NAH):
    - Production X-CLIENT-ID
    - Production Secret Key / API Key
    - Production Public key of NAH
  - Credentials to prepare from your end:
    - Your Production Webhook URL
    - Your Production Private & Public Key
- Request for production configuration to NAH. \
  To setup your sandbox settings, you can send an email to \<TBC Tech Onboarding Email\> with the following Subject: "Request for production configuration". The request should include
  - Production Public Key to be shared to us in encrypted file to \<TBC Tech Onboarding Email\>. Password to be shared in different channel / same channel (i.e. Email, Whatsapp, Telegram) with different PIC
  - Production Webhook URL
  - IP Whitelist for production access (only registered IP address can access your account via API)

## Credentials Sharing

To share credentials to your end, NAH will share the following information to you via admin/ first registered account on our end.

- The credentials will need to be encrypted with NAH's public key and signed with Merchant's private key.
- NAH will verify the signature of the credentials using merchant's shared public key

In order to do this, you will need to share your public key to NAH, and NAH will share NAH's public key to your side.
